# 🎅 Santa Point App 仕様書 v7.0（最終版）
Last updated: 2026-03-07
Status: ✅ 確定・実装準備完了

---

## 1. コンセプト

3〜7歳の子どもが、自分だけのサンタさんに**通年を通して**毎日話しかけてポイントを貯め、プレゼントやメダルを集めていくアプリ。

**最大の特徴**：オンボーディング時に100人の中からランダムで「担当サンタ」が決まり、毎年同じサンタが成長を見守り続ける。子どもとサンタの4年間にわたる絆が生まれる。

クリスマスは1年の集大成イベントだが、**365日いつでも楽しめる**のがコアバリュー。ホーム画面を中心にモーダル・ウィジェットで完結するシンプルな設計。難しい操作なし・継続プレッシャーなし・ただ楽しい。

---

## 2. 対象ユーザー

- **子ども**：3〜7歳（複数人対応・タブ切り替え）
- **親**：オンボーディング設定・プレゼント交換判断・データ管理を担当
- **利用期間**：最大4年間（通年使用・毎年クリスマスに集大成）

---

## 3. デザイン方針

- 世界観：ダーク背景＋クリスマス（深い紫・赤・金）
- メインビジュアル：アイソメトリッククレイ調のサンタのお部屋（3D感・暖かい光）
- フォント・文体：ひらがな多め・3〜7歳が読める表現
- UI：大きなボタン・モーダル中心・直感操作
- アニメーション制作ツール：**nanobanana / Kling.ai** を活用

## 6. オンボーディング画面

### Step 1：開始方法の選択
- 「🎄 あたらしくはじめる」
- 「📦 データをひきついではじめる」

### Step 2：子どもの情報入力（新しく始める選択時）
- 入力項目：**名前**・**生年月日**のみ
- ひらがな入力対応

### Step 3：担当サンタの決定演出
- 「あなただけのサンタさんをさがしています...」アニメーション
- 名前・生年月日をもとに100人の中からランダムで1人が決まる
- 担当サンタが登場する演出（ドン！と出てくる）
- 担当サンタの名前・ひとことが表示される
  - 例：「まほうつかいのサンタです！よろしくね🪄」
- **このサンタは毎年同じ**（4年間担当し続ける・子どもの成長を見守る）

### データ引き継ぎ選択時
- 「ファイルをよみこむ」ボタンをタップ
- エクスポートしたJSONファイルを選択してインポート
- データが復元されたらホームへ

## 7. ホーム画面

ホームが全ての起点。すべての操作はここから始まる。

### 7-1. プロフィールタブ（上部）
- 子どもごとのタブ切り替え（例：ゆうくん・はなちゃん）
- ＋ボタンで子どもを追加

| バッジ | 内容 | 備考 |

| 🎄 XX日まで | クリスマスまでの残り日数 | 通年表示 |

### 7-3. お手紙バナー（条件表示）
- 未読のお手紙がある場合のみ表示（赤いドットなし）
- 子ども向け：「📩 サンタさんからおてがみだよ！」
- 親向けパターン：「📩 ママ・パパにおてがみだよ！」
- タップでお手紙モーダルが開く

### 7-4. メインビジュアル（サンタのお部屋）

### 7-5. ウィッシュリスト（下部）
- 「おねがいしたもの」セクション

- チャット画面で編集可能

### 7-6.「サンタさんとはなす」固定ボタン
- 画面下部に常に表示される赤い大ボタン
- タップで「サンタにほうこく」モーダルが開く

## 8. サンタにほうこく（モーダル）

チャット形式のUI。音声入力後にテキスト編集が可能。

### 画面構成
- 上部：担当サンタのクレイ調アバター
- 中央：チャット領域
  - 子どもの発言（右寄せ・ピンク背景）
  - サンタの返答（左寄せ・白背景）
  - ポイント付与メッセージ＋「+Xポイントゲット！」バッジ
- 下部：
  - テキスト入力欄（音声認識後に編集可能）
  - マイクボタン（赤・大）
  - 「タップしてサンタさんに話しかけよう！」テキスト

### 音声→テキスト編集フロー
1. マイクボタンをタップ → 音声入力開始
2. 話しかける（「はみがきしたよ！」など）
3. 音声がテキストに変換されて入力欄に表示
4. **送信前にテキスト編集が可能**
5. 送信ボタンでメッセージ送信
6. Claude APIでサンタキャラの返答を生成（担当サンタの個性・季節・イベントを反映）
7. 返答表示＋TTS音声で流れる
8. **1〜3ptをランダム付与**（優劣なし・内容不問）
9. 「+Xポイントゲット！」バッジ表示

### ポイント付与ルール
- 報告内容に関わらず **1・2・3pt のいずれかをランダムで付与**
- 優劣なし・ミッション概念なし・継続ボーナスなし
- シンプルに「話しかけたら必ずもらえる」

## 9. サンタからのおてがみ（モーダル）

### 画面構成
- 最新のお手紙：大きなカード形式
  - 「あたらしい！」バッジ（未読時）
  - 日付・タイトル・宛名・本文・署名「サンタクロースより」（赤字）
- 「これまでのおてがみ」：過去のお手紙リスト

### お手紙の種類

| 種別 | 宛先 | タイミング |
|------|------|-----------|
| 子ども向け（通常） | 子ども | 月1回ペース・通年 |
| 子ども向け（イベント） | 子ども | 誕生日・クリスマス・特別イベント時 |
| 親向け | ママ・パパ | ポイント状況報告・プレゼント提案など |

### 生成
- Claude APIでパーソナライズ生成（名前・ポイント状況・担当サンタのキャラ・季節・直近のイベントに基づく）
- ひらがな多め・3〜7歳向け文体

## 10. 🏅 サンタメダル図鑑（モーダル）

ポイントとは別の**長期コレクション**要素。4年かけてコンプリートを目指す。

### 概要
- 全100人のユニークなサンタキャラクターを収録
- **通年累計ポイント**が一定数に到達するたびにメダルが1枚付与
- 1年では全メダルは集まらない設計（4年間の楽しみ）
- どのサンタが当たるかはランダム（お楽しみ）
- 図鑑UIで一覧確認。未取得は影のシルエット表示

### 通年累計ポイントによるメダル付与

| 〜1,500pt | 〜50枚（3年目） |

## 11. 🎁 プレゼント交換の仕組み

### ポイントの二重管理

| 種別 | リセット | 用途 |
|------|---------|------|
| **今年のポイント** | 毎年1/1にリセット | プレゼント交換の判断軸 |
| **通年累計ポイント** | リセットなし・一生積み上がる | メダル付与の軸 |

### プレゼント交換フロー
1. 子どもが音声でウィッシュリストにお願いを登録
2. 年間を通じてポイントが貯まる
3. **親が任意のタイミングで交換を承認**（必ずもらえる設計）
4. クリスマスが年間の集大成・大型プレゼント交換の機会
5. クリスマス以外でも親の判断でいつでも交換可能

### 年間ポイントのリセット
- 毎年1/1に今年のポイントがリセット
- リセット前に通年累計へ加算
- 「去年はXXpt貯めたね！」という振り返り表示も可能

## 15. ⚙️ 設定画面

ホームの⚙️ボタンからアクセスできる唯一の独立画面。**親が操作する想定**。

### 設定項目

| セクション | 内容 |
|-----------|------|
| 👧 こどものついか | 新しい子どものプロフィールを追加（名前・生年月日） |
| 📤 データをエクスポート | 全データをJSONファイルとして端末に書き出し |
| 📥 データをインポート | JSONファイルを読み込んでデータを復元 |
| 🔗 かんれんサービス | ElevenLabs / OpenAI APIキー設定など |
| 📄 ほうてきじょうほう | プライバシーポリシー・利用規約 |

### データ引き継ぎの具体的な流れ

**エクスポート（旧端末）**
1. 設定画面 →「データをエクスポート」
2. `santa_point_backup_YYYYMMDD.json` として端末に保存
3. AirDrop・メール・クラウドストレージなど任意の方法で新端末へ転送

**インポート（新端末）**
1. オンボーディングで「データをひきついではじめる」を選択、または設定画面 →「データをインポート」
2. JSONファイルを選択
3. 子どもの情報・今年のポイント・累計ポイント・メダル・担当サンタ・ウィッシュリスト・お手紙履歴がすべて復元

### 保存データの構造（JSON）

## 16. 🏠 サンタのお部屋・リアルタイム連動仕様

### 概要
端末の**現在日時を取得**し、時間帯・季節・特別日付に応じてお部屋の見た目とサンタの行動が自動的に変化する。毎日開くたびに違う様子が楽しめる。

---

### 16-1. 時間帯による変化

| 時間帯 | お部屋の様子 | サンタの状態 |
|-------|------------|------------|
| 深夜（0〜5時） | 部屋が暗い・月明かり・静か | ぐっすり眠っている（いびきアニメ） |
| 朝（6〜8時） | 朝日が差し込む・カーテンが少し開く | まだ眠そう・目をこすっている |
| 午前（9〜11時） | 明るい室内・朝ごはんの気配 | 朝ごはんを食べている・コーヒーを飲んでいる |
| 昼（12〜13時） | 窓から明るい光 | お昼寝している or 外出中（トナカイと散歩） |
| 午後（14〜17時） | 落ち着いた室内の光 | プレゼントを作っている・本を読んでいる |
| 夕方（18〜19時） | 暖炉の光が強くなる・夕焼け | お茶を飲んでいる・くつろいでいる |
| 夜（20〜22時） | 暖炉の炎・ランプの光 | パジャマに着替えている |
| 深夜前（23時〜） | 部屋が暗くなっていく | 歯磨き中・就寝準備 |

---

### 16-2. 季節による変化（フィクション設定）

| 季節 | 時期 | お部屋・お庭の様子 | サンタの様子 |
|------|------|-----------------|------------|
| 冬（クリスマスシーズン） | 12月 | 雪・クリスマスデコ全開・暖炉フル稼働 | 忙しそうに準備・プレゼントが増えていく |
| 春 | 3〜5月 | 窓の外に花が咲く・雪が溶けている | トナカイと外遊び・ガーデニング |
| 夏 | 6〜8月 | 窓から緑の木々・明るい光 | アイスを食べている・水遊び・昼寝 |
| 秋 | 9〜11月 | 紅葉・落ち葉・かぼちゃ | プレゼントのリストを作り始める・読書 |

---

### 16-3. 特別日付による変化

| 日付 | 演出 |
|------|------|
| 子どもの誕生日 | サンタがお祝いしてくれる演出・特別メッセージ |
| 1/1 | サンタが着物・お正月飾り |
| 5/5 | サンタが兜をかぶっている |
| 7/7 | サンタが短冊を書いている |
| 10/31 | サンタが変装している（担当サンタの個性で変装が変わる） |
| 12/1 | シーズン開幕・デコレーションが一気に増える |
| 12/20〜23 | 荷造りで忙しそう・プレゼント袋が満杯に |
| 12/24 | ソリで出発する大演出アニメーション🎅 |
| 12/25 | 旅から帰ってきてぐったり寝ている |

---

### 16-4. アニメーション制作方針

- **nanobanana**：クレイ調キャラクターの静止画・アイソメトリック部屋の素材制作
- **Kling.ai**：サンタの動きアニメーション（眠る・食べる・作業する等）の生成
- 各シーン・状態ごとにLoopアニメーションを用意
- Lottie形式でアプリに組み込み（React Native対応）
- 状態の切り替えはクロスフェードで滑らかに遷移

---

### 16-5. 不在パターン

- 週に1〜2回程度、ランダムでお出かけ中
- 不在時はお部屋にメモが置かれている（「トナカイとそりのれんしゅうにいってます」など）
- 午後〜夕方の時間帯に発生しやすい設定

### 実装メモ

- **担当サンタ決定ロジック**：`(名前の文字コード合計 + 生年月日の数字合計) % 100 + 1` でNo.を決定（同じ子は常に同じサンタ・再現性あり）
- **ビジュアル制作**：nanobananaで各サンタのクレイ調画像を生成 → Kling.aiでアニメーション化 → Lottie形式でアプリ組み込み
- **メダル付与**：累計ポイント到達時に未取得のNo.からランダムで1枚付与
- **図鑑表示**：取得済みはフルカラー、未取得はシルエット＋「？」
- **Claude APIへの反映**：担当サンタのNo.・名前・ひとことをシステムプロンプトに含め、返答の口調・個性を反映
